Important Terms/Concepts
  • Tag - syntax/mark up language to define what and/or how content is being formatted.
  • Element - contained inside the tag structure. The term also refers to the entire structure of the tag and its contents
  • Attribute - provides additional information, located in the start tag attribute = "value"
  • Inheritance - certain properties from child are directly inherited from their parent element like div elements having a color property; this can be overridden if done so through classes
  • selector/CSS rule - a targeting element for CSS rules; for multiple selectors of the same rules use commas for separation;
    if hierarchies, dont use commas use spaces or dot notation (brackets?)
  • property:value => goes in your selectors
  • Viewport - your electronic device screen
  • Window - browser window
  • document - webpage
  • What is a computer? Really?: Machines that help us solve problems by manipulating information
      All computers take input, store information, process it, and then output.
    • Input - A computer listens for something that happens to start doing an action. These can literally be input devices like your keyboard or a touch screen.
    • Store Information - Computer store the inputs into memory/data
    • Process - A processor would take information from memory and manipulate it using an algorithm, through a series of commands. It places this processed information back into memory. This is repeated until we get the desired output (process the input)
    • Output - An output can be really anything, dependent on what you're trying to display with your computer. A display can show text, photos, video for example. (the information processed by the processor (function))

      • In Javascript, we can demonstrate these concepts as well:

      Input - Event Listeners can be treated like input devices. They listen for an event to happen and run a function

      Store Information - Programming languages like Javascript already has a built in way to store information through variables

      Process - We can also use functions to manipulate information

      Output - We can also use functions to show information. These outputs in regards to Javascript, is updating out DOM, or logging the console, even sending out API calls, for example.

      If say we had a function to increment a number. The input would be our number, we would store the number in a variable, the processes would increment the number and place it back into the variable (memory), and finally we would return the value of the variable (output)

    Color Theory
    
      Two colors that are opposite from each other on the color wheel are called complementary colors. 
        If two complementary colors are combined, they produce gray. 
        But when they are placed side-by-side, these colors produce strong visual contrast and appear brighter.
    
      
    
      
    
      RGB Color Model
      
    
      3 parameters representing red, green, and blue value. They range from 0 to 255. Half range (127) gets tierary colors. 
      
    And combining any combo of the 3 colors gets you secondary colors, with the range varying the color shades.
      
    
    
      
    
      Hex Colors
      
    
      Hex color values start with a # character and take six characters from 0-9 and A-F (in that order). 
        The first pair of characters represent red, the second pair represent green, 
        and the third pair represent blue. For example, #4B5320. (there is an extra pair for alpha channel)
    
      
    
      00 is 0% of that color, and FF is 100%
      

    
      
    
      HSL Color Model (Hue Saturation Lightness)
      
    
      The CSS hsl function accepts 3 values: a number from 0 to 360 for hue, a percentage from 0 to 100 for saturation, and 
        a percentage from 0 to 100 for lightness.
      
    
      If you imagine a color wheel, the hue red is at 0 degrees, green is at 120 degrees, and blue is at 240 degrees.
      
    
    
      Saturation is the intensity of a color from 0%, or gray, to 100% for pure color.
      
    
      Lightness is how bright a color appears, from 0%, or complete black, to 100%, complete white, with 50% being neutral.

        When would you clone a repository versus fork a repository? 
        
            If you're working with a repository and have write access, you can clone it to your local machine. 
        
            From there, you can make modifications and push your changes directly to the origin repository. 
        
            If you need to work with a repository created by another owner such as github/example and don't have write access, 
        
            you can fork the repository into your GitHub account, then clone the fork to your local machine. 
        
            You can then make changes to your local copy of githubtraining/example, 
        
            then push them back to your remote origin repository (githubtraining/example). 
        
            You can then submit these changes to the github/example upstream repository using a pull request.
